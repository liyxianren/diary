第四节：实战项目

实战项目1：贪吃蛇游戏

项目概述

通过这个项目，你将学会如何使用 Claude Code 从零开始创建一个完整的网页游戏，包括游戏逻辑、UI 设计和用户交互。

⏱️ 预计时间：30 分钟 📝 难度：入门 🛠️ 技术：HTML + CSS + JavaScript

功能要求

  • ✅ 贪吃蛇可以通过方向键控制移动
  • ✅ 吃到食物后蛇身变长，分数增加
  • ✅ 撞墙或撞到自己游戏结束
  • ✅ 显示当前分数和最高分
  • ✅ 支持暂停和重新开始
  • ✅ 响应式设计，适配不同屏幕

实战步骤

1

创建项目文件夹

mkdir snake-game
cd snake-game
claude
2

创建 CLAUDE.md 配置

告诉 Claude Code 这个项目的要求：

创建 CLAUDE.md 文件，内容如下：

# 贪吃蛇游戏项目

## 技术栈
- 纯原生 HTML + CSS + JavaScript
- 不使用任何框架或库
- Canvas API 绘制游戏画面

## 文件结构
- index.html（游戏页面）
- style.css（样式）
- game.js（游戏逻辑）

## 编码规范
- 使用 ES6+ 语法
- 函数名使用 camelCase
- 常量使用 UPPER_SNAKE_CASE
- 添加注释说明关键逻辑

## 游戏参数
- 网格大小：20x20
- 蛇初始长度：3
- 移动速度：150ms
- 每吃一个食物加 10 分
3

使用 Prompt 创建游戏

现在给 Claude Code 一个清晰的 Prompt：

创建一个贪吃蛇游戏，包含以下功能：

1. 游戏界面
   - 使用 Canvas 绘制 20x20 的游戏区域
   - 蛇用绿色方块表示，食物用红色圆点表示
   - 显示当前分数和最高分（使用 localStorage）
   - 添加开始、暂停、重新开始按钮

2. 游戏逻辑
   - 蛇初始在中心，长度为 3
   - 使用方向键控制蛇的移动方向
   - 蛇每 150ms 移动一格
   - 吃到食物后：蛇身+1，分数+10，生成新食物
   - 游戏结束条件：撞墙或撞到自己的身体

3. UI 设计
   - 现代化扁平风格
   - 使用渐变背景
   - 按钮有 hover 效果
   - 游戏结束显示弹窗

4. 代码要求
   - 代码结构清晰，添加注释
   - 使用面向对象的方式组织代码
   - 所有常量定义在文件开头

请创建 index.html、style.css、game.js 三个文件
4

测试和调整

Claude Code 会自动创建所有文件。测试游戏：

# 在浏览器中打开
open index.html  # Mac
start index.html # Windows

如果有问题，可以继续对话：

  • "游戏速度太快，改为 200ms"
  • "添加音效，吃到食物时播放提示音"
  • "添加难度选择：简单/普通/困难"

实战项目2：待办事项应用

项目概述

创建一个功能完整的待办事项应用，学习 CRUD 操作、本地存储和现代 UI 设计。

⏱️ 预计时间：40 分钟 📝 难度：进阶 🛠️ 技术：React + TypeScript

功能要求

  • ✅ 添加、编辑、删除待办事项
  • ✅ 标记任务完成/未完成
  • ✅ 按状态筛选（全部/进行中/已完成）
  • ✅ 搜索功能
  • ✅ 数据持久化（localStorage）
  • ✅ 拖拽排序
  • ✅ 深色模式切换

快速开始 Prompt

创建一个 React + TypeScript 的待办事项应用

技术要求：
- 使用 Vite 创建项目
- TypeScript 严格模式
- 使用 React Hooks
- CSS Modules 样式隔离

功能清单：
1. 任务管理
   - 添加新任务（输入框 + 添加按钮）
   - 编辑任务（双击进入编辑模式）
   - 删除任务（删除按钮 + 确认对话框）
   - 标记完成/未完成（复选框）

2. 筛选和搜索
   - 三个筛选按钮：全部/进行中/已完成
   - 搜索框：实时搜索任务标题
   - 显示任务计数

3. 数据持久化
   - 使用 localStorage 保存数据
   - 页面加载时自动恢复数据
   - 每次操作后自动保存

4. UI/UX
   - Material Design 风格
   - 深色/浅色模式切换
   - 响应式设计
   - 动画过渡效果
   - 空状态提示

5. 高级功能
   - 任务优先级（高/中/低）
   - 截止日期选择
   - 拖拽排序（使用 react-beautiful-dnd）
   - 批量操作

请创建完整的项目结构，包括组件、类型定义、样式文件

实战技巧总结

💡 开发技巧

  • 从简单开始：先实现核心功能，再逐步添加特性
  • 迭代开发：不要一次性要求所有功能，分步骤完成
  • 充分测试：每完成一个功能就测试，及时发现问题
  • 代码审查：让 Claude Code 审查自己生成的代码，找出潜在问题

🎯 Prompt 技巧

  • 使用模板：参考前面的 Prompt 模板，形成自己的风格
  • 提供示例：给出期望的输入输出示例
  • 明确约束：说明什么该做，什么不该做
  • 分步执行：复杂任务拆分为多个小任务

⚠️ 常见陷阱

  • 避免：模糊的需求描述
  • 避免：一次性要求太多功能
  • 避免：不提供技术栈信息
  • 避免：忘记配置 CLAUDE.md

🚀 效率提升

  • 使用 Plan 模式：大型项目先规划再执行
  • 创建模板：常用的项目结构保存为模板
  • 善用 rules：团队规范写入 .claude/rules/
  • 快捷键：熟记 Alt+M 等常用快捷键
